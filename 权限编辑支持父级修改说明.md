### 3. 核心算法 - 路径查找

```javascript
// 从树状数据中查找节点路径（递归查找）
const findNodePath = (tree, targetId, currentPath = []) => {
  if (!tree || tree.length === 0) return null
  
  for (const node of tree) {
    // 找到目标节点，返回当前路径
    if (node.id === targetId) {
      return [...currentPath, node.id]
    }
    
    // 如果有子节点，递归查找
    if (node.childrenPermission && node.childrenPermission.length > 0) {
      const found = findNodePath(
        node.childrenPermission, 
        targetId, 
        [...currentPath, node.id]
      )
      if (found) return found
    }
  }
  
  return null
}
```

**算法说明**：
- 递归遍历树状数据
- 找到目标节点时返回完整路径
- 使用深度优先搜索（DFS）
- 时间复杂度：O(n)
- 空间复杂度：O(h)，h 为树的高度

## 💻 后端实现参考

### Service 层实现

#### 1. 获取权限树实现

```java
/**
 * 获取启用的权限树
 * @return 树状结构的权限列表
 */
public List<PermissionTreeResponse> selectTreeByStatus() {
    // 查询所有启用的权限
    List<Permission> allPermissions = lambdaQuery()
        .eq(Permission::getStatus, true)
        .orderByAsc(Permission::getSort)
        .list();
    
    // 构建树状结构
    return buildTree(allPermissions, 0L);
}

/**
 * 递归构建树
 * @param permissions 所有权限列表
 * @param parentId 父级ID
 * @return 树状结构
 */
private List<PermissionTreeResponse> buildTree(List<Permission> permissions, Long parentId) {
    List<PermissionTreeResponse> tree = new ArrayList<>();
    
    for (Permission permission : permissions) {
        if (permission.getParentId().equals(parentId)) {
            PermissionTreeResponse node = new PermissionTreeResponse();
            BeanUtils.copyProperties(permission, node);
            
            // 递归查找子节点
            List<PermissionTreeResponse> children = buildTree(permissions, permission.getId());
            if (!children.isEmpty()) {
                node.setChildrenPermission(children);
            }
            
            tree.add(node);
        }
    }
    
    return tree;
}
```

## ⚠️ 注意事项

### 1. 循环引用检查
后端在更新父级时需要检查，避免循环引用：
```java
/**
 * 检查是否会产生循环引用
 */
private boolean checkCircularReference(Long id, Long newParentId) {
    if (newParentId == null || newParentId == 0) {
        return false;
    }
    
    // 检查新父级是否是当前节点的子孙节点
    List<Long> path = getParentPath(newParentId);
    return path.contains(id);
}
```

### 2. 权限验证

## 📊 数据库设计建议

### 推荐字段
```sql
CREATE TABLE permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL COMMENT '权限名称',
    code VARCHAR(100) NOT NULL COMMENT '权限标识',
    type TINYINT NOT NULL COMMENT '类型 1-菜单 2-按钮',
    path VARCHAR(200) COMMENT '路由路径',
    parent_id BIGINT DEFAULT 0 COMMENT '父级ID',
    status BOOLEAN DEFAULT TRUE COMMENT '状态',
    sort INT DEFAULT 0 COMMENT '排序',
    level INT DEFAULT 0 COMMENT '层级深度（可选，用于优化查询）',
    path_ids VARCHAR(500) COMMENT '完整路径IDs（可选，用于优化查询，如：1,3,5）',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_parent_id (parent_id),
    INDEX idx_type (type),
    INDEX idx_status (status)
) COMMENT '权限表';
```

### 可选优化字段说明
- `level`: 存储节点深度，可以快速判断层级
- `path_ids`: 存储完整路径字符串，避免递归查询
  - 例如：`1,3,5` 表示路径 [1, 3, 5]
  - 需要在更新父级时同步更新
