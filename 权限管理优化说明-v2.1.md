# 权限管理优化说明 v2.1

## ✨ 优化内容

### 移除 `/path/{id}` 接口依赖

**优化前**：需要两个接口

## 🎯 核心改进

### 1. 本地路径计算

**新增递归查找算法**：
```javascript
// 从树状数据中查找节点路径
const findNodePath = (tree, targetId, currentPath = []) => {
  if (!tree || tree.length === 0) return null
  
  for (const node of tree) {
    // 找到目标节点，返回当前路径
    if (node.id === targetId) {
      return [...currentPath, node.id]
    }
    
    // 递归查找子节点
    if (node.childrenPermission && node.childrenPermission.length > 0) {
      const found = findNodePath(
        node.childrenPermission, 
        targetId, 
        [...currentPath, node.id]
      )
      if (found) return found
    }
  }
  
  return null
}
```

**算法特点**：
- 深度优先搜索（DFS）
- 时间复杂度：O(n)
- 空间复杂度：O(h)，h为树高度
- 即时响应，无网络延迟

### 2. 使用方式

```javascript
// 编辑权限时
const handleEdit = async (row) => {
  // 1. 加载权限树
```

## 🔧 后端只需实现

### 唯一必需接口

```java
/**
 * 获取启用的权限树
 * @return 树状结构的权限列表
 */
@GetMapping("/select")
public Result<List<PermissionTreeResponse>> select() {
    return Result.success(permissionService.selectTreeByStatus());
}
```

### 返回数据结构

```json
[
  {
    "id": 1,
    "name": "系统管理",
    "type": 1,
    "parentId": 0,
    "status": true,
    "sort": 1,
    "childrenPermission": [
      {
        "id": 2,
        "name": "用户管理",
        "type": 1,
        "parentId": 1,
        "status": true,
        "sort": 1,
        "childrenPermission": []
      }
    ]
  }
]
```

**关键要求**：
- 只返回 `status = true` 的权限
- 使用 `childrenPermission` 字段存储子级
- 递归构建完整树状结构
- 按 `sort` 字段排序
