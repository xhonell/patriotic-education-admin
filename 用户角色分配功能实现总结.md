## 🔌 API 接口

### 已实现的 API 函数

```javascript
// src/api/user-role.js

// 1. 获取用户角色列表
export function getUserRoles(userId)

// 2. 分配角色（替换模式）
export function assignRoles(data)

// 3. 添加单个角色
export function addUserRole(userId, roleId)

// 4. 删除单个角色
export function removeUserRole(userId, roleId)

// 5. 删除所有角色
export function removeAllUserRoles(userId)
```

### 后端接口映射

| 前端函数 | HTTP 方法 | 后端路径 | 权限 |
|---------|---------|---------|------|
| `getUserRoles` | GET | `/user-role/list/{userId}` | `admin:user-role:list` |
| `assignRoles` | POST | `/user-role/assign` | `admin:user-role:assign` |
| `addUserRole` | POST | `/user-role/add/{userId}/{roleId}` | `admin:user-role:add` |
| `removeUserRole` | DELETE | `/user-role/remove/{userId}/{roleId}` | `admin:user-role:remove` |
| `removeAllUserRoles` | DELETE | `/user-role/remove-all/{userId}` | `admin:user-role:remove-all` |

## 🔄 工作流程

### 完整的角色分配流程

```
1. 用户进入用户列表页面
   ↓
2. 点击某用户的"分配角色"按钮
   ↓
3. 打开 UserRoleDialog 对话框
   ↓
4. 对话框并行加载：
   - 用户当前角色列表 (getUserRoles)
   - 系统所有可用角色 (getRoleList)
   ↓
5. 显示加载结果：
   - 当前角色（绿色标签，可移除）
   - 可选角色（下拉框多选）
   ↓
6. 用户操作：
   - 移除不需要的角色（点击标签 ×）
   - 选择新的角色（下拉框多选）
   ↓
7. 点击"确定分配"按钮
   ↓
8. 调用 assignRoles API：
   - 发送 POST 请求到 /user-role/assign
   - 请求体: { userId, roleIds }
   ↓
9. 后端处理：
   - 删除用户原有的所有角色
   - 分配新的角色
   - 返回成功响应
   ↓
10. 前端处理：
    - 显示"角色分配成功"提示
    - 关闭对话框
    - 刷新用户列表
    ↓
11. 返回用户列表页面
    - 用户的角色已更新
```

## 🚀 性能优化

### 1. 并行加载

使用 `Promise.all()` 并行加载用户角色和所有角色，减少加载时间：

```javascript
const [rolesRes, allRolesRes] = await Promise.all([
  getUserRoles(props.userData.id),
  getRoleList({ page: 1, pageSize: 1000 })
])
```

**性能提升**: 从 2 个串行请求 (2s) 优化为 1 个并行请求 (1s)

### 2. 计算属性缓存

使用 `computed` 自动缓存计算结果，避免重复计算：

```javascript
const availableRoles = computed(() => {
  return allRoles.value.filter(role => 
    !currentRoles.value.some(current => current.roleId === role.id)
  )
})
```

**性能提升**: 减少不必要的过滤操作

### 3. 去重处理

使用 `Set` 自动去重，避免重复的角色分配：

```javascript
const uniqueRoleIds = [...new Set(allRoleIds)]
```

**性能提升**: 减少数据库操作

### 4. 虚拟滚动（可选）

如果角色数量超过 1000，可使用虚拟滚动优化列表性能：

```javascript
// 使用 el-virtual-list 替代普通列表
<el-virtual-list :items="roles" :item-size="50">
  <template #default="{ item }">
    <!-- 角色项 -->
  </template>
</el-virtual-list>
```
