# 角色权限分配功能说明

## 📋 功能概述

在角色管理的基础上，新增了权限分配功能。在创建或编辑角色时，可以为角色分配系统权限，实现基于角色的权限控制（RBAC）。

## ✨ 新增特性

### 1. 权限树形选择
- 使用 Element Plus Tree 组件
- 展示完整的权限树状结构
- 支持全选、半选状态
- 父子节点联动选择
- 区分菜单和按钮类型

### 2. 可视化展示
- **菜单权限**：蓝色图标 + 蓝色标签
- **按钮权限**：橙色图标 + 橙色标签
- 清晰的层级结构
- 默认全部展开

### 3. 智能处理
- 自动包含半选中的父节点
- 编辑时自动回显已选权限
- 异步加载权限树
- 空状态友好提示

## 📊 数据结构变更

### 角色实体（Role）新增字段

```java
/**
 * 角色关联的权限ID列表
 */
private List<Long> permissionIds;
```

完整的 Role 实体：
```java
{
  id: Long,                    // 主键
  name: String,                // 角色名称
  description: String,         // 角色描述
  status: Boolean,             // 状态（true 启用，false 禁用）
  permissionIds: List<Long>,   // 权限ID列表（新增）
  createTime: LocalDateTime,   // 创建时间
  updateTime: LocalDateTime    // 更新时间
}
```

### 权限选择逻辑

**1. 新增角色**
```

2. 对话框打开
3. 异步加载权限树
4. 显示所有启用的权限
5. 用户勾选需要的权限
6. 提交时自动收集：
   - 完全选中的节点
   - 半选中的父节点
```

**2. 编辑角色**
```
1. 点击"编辑"按钮
2. 对话框打开，填充角色信息
3. 异步加载权限树
4. 根据 permissionIds 自动勾选对应节点
5. 用户修改权限
6. 提交时更新权限列表
```

**3. 提交处理**
```javascript
// 获取完全选中的节点
const checkedKeys = permissionTreeRef.value.getCheckedKeys()

// 获取半选中的父节点（重要！）
const halfCheckedKeys = permissionTreeRef.value.getHalfCheckedKeys()

// 合并所有权限ID
form.permissionIds = [...checkedKeys, ...halfCheckedKeys]
```

> **为什么要包含半选中节点？**
> 当用户只选中某个菜单下的部分按钮时，该菜单处于半选中状态。
> 但该菜单本身也是一个权限，需要包含在权限列表中。

## 🔧 技术实现

### 前端核心代码

**1. 加载权限树**
```javascript
const loadPermissionTree = async () => {
  permissionLoading.value = true
  try {
    const res = await getPermissionSelectTree()
    permissionTree.value = res || []
  } catch (error) {
    console.error(error)
    ElMessage.error('加载权限列表失败')
    permissionTree.value = []
  } finally {
    permissionLoading.value = false
  }
}
```

**2. 编辑时回显权限**
```javascript
const handleEdit = async (row) => {
  // 1. 填充表单
  Object.assign(form, {
    id: row.id,
    name: row.name,
    description: row.description,
    status: row.status,
    permissionIds: row.permissionIds || []
  })
  
  // 2. 打开对话框
  dialogVisible.value = true
  
  // 3. 加载权限树
  await loadPermissionTree()
  
  // 4. 设置选中状态（延迟执行，确保树已渲染）
  if (permissionTreeRef.value && form.permissionIds.length > 0) {
    setTimeout(() => {
      permissionTreeRef.value.setCheckedKeys(form.permissionIds)
    }, 100)
  }
}
```

**3. 提交权限数据**
```javascript
const handleSubmit = async () => {
  await formRef.value.validate()
  
  // 收集所有选中的权限（包括半选中）
  const checkedKeys = permissionTreeRef.value.getCheckedKeys()
  const halfCheckedKeys = permissionTreeRef.value.getHalfCheckedKeys()
  form.permissionIds = [...checkedKeys, ...halfCheckedKeys]
  
  const apiFunc = form.id ? updateRole : addRole
  await apiFunc(form)
  
  ElMessage.success(form.id ? '更新成功' : '新增成功')
  dialogVisible.value = false
  loadData()
}
```

**4. 重置表单**
```javascript
const resetForm = () => {
  form.id = null
  form.name = ''
  form.description = ''
  form.status = true
  form.permissionIds = []
  formRef.value?.clearValidate()
  
  // 清空树的选中状态
  if (permissionTreeRef.value) {
    permissionTreeRef.value.setCheckedKeys([])
  }
}
```

### 后端实现要点

**1. 保存角色时处理权限关联**
```java
@PostMapping("/save")
public Result<String> save(@RequestBody Role role) {
    // 1. 保存角色基本信息
    roleService.saveBy(role);
    
    // 2. 保存角色权限关联（需要实现）
    // 通常是往 role_permission 关联表插入数据
    if (role.getPermissionIds() != null && !role.getPermissionIds().isEmpty()) {
        roleService.saveRolePermissions(role.getId(), role.getPermissionIds());
    }
    
    return Result.success();
}
```

**2. 更新角色时处理权限关联**
```java
@PostMapping("/update")
public Result<String> update(@RequestBody Role role) {
    // 1. 更新角色基本信息
    roleService.updateBy(role);
    
    // 2. 更新角色权限关联
    // 先删除旧的关联，再插入新的关联
    if (role.getPermissionIds() != null) {
        roleService.deleteRolePermissions(role.getId());
        if (!role.getPermissionIds().isEmpty()) {
            roleService.saveRolePermissions(role.getId(), role.getPermissionIds());
        }
    }
    
    return Result.success();
}
```

**3. 查询角色列表时包含权限**
```java
@GetMapping("/list")
public Result<PageInfo<Role>> list(RolePageRequest request) {
    PageInfo<Role> pageInfo = roleService.selectList(request);
    
    // 为每个角色填充权限ID列表
    for (Role role : pageInfo.getList()) {
        List<Long> permissionIds = roleService.getPermissionIdsByRoleId(role.getId());
        role.setPermissionIds(permissionIds);
    }
    
    return Result.success(pageInfo);
}
```

**4. 数据库表结构**

**角色表（pe_role）**：
```sql
CREATE TABLE pe_role (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(50) NOT NULL COMMENT '角色名称',
  description VARCHAR(200) COMMENT '角色描述',
  status BOOLEAN DEFAULT TRUE COMMENT '状态',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**角色权限关联表（pe_role_permission）**：
```sql
CREATE TABLE pe_role_permission (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  role_id BIGINT NOT NULL COMMENT '角色ID',
  permission_id BIGINT NOT NULL COMMENT '权限ID',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_role_permission (role_id, permission_id)
);
```

## ⚠️ 注意事项

1. **权限树来源**
   - 使用 `/admin/permission/select` 接口
   - 只显示 `status=true` 的权限
   - 树形结构通过 `childrenPermission` 字段组织

2. **半选中节点**
   - 必须包含半选中的父节点
   - 否则权限树会不完整
   - 使用 `getHalfCheckedKeys()` 方法获取

3. **回显时机**
   - 必须等待权限树加载完成后再设置选中
   - 使用 `setTimeout` 延迟 100ms 确保渲染完成
   - 否则可能无法正确回显

4. **权限验证**
   - 后端需要验证权限ID的有效性
   - 防止传入已禁用或不存在的权限ID

5. **事务处理**
   - 保存角色和权限关联应在同一事务中
   - 确保数据一致性

6. **性能优化**
   - 权限树数据可考虑缓存
   - 大量权限时可考虑懒加载
   - 当前实现为一次性加载全部
