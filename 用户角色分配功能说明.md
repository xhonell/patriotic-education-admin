# 用户角色分配功能说明

## 📋 功能概述

用户角色分配功能允许管理员为平台用户分配一个或多个角色，实现灵活的权限管理。该功能与后端 `/user-role` 接口完全对接。

```
└── views/

```

## 🔌 API 接口

### 1. 获取用户的角色列表

```javascript
import { getUserRoles } from '@/api/user-role'

// 使用方式
const roles = await getUserRoles(userId)

// 返回数据结构
[
  {
    id: 1,                    // 关系ID
    userId: 1,                // 用户ID
    roleId: 1,                // 角色ID
    roleName: "管理员",       // 角色名称
    roleDescription: "系统管理员",  // 角色描述
    roleStatus: true,         // 角色状态
    createTime: "2025-12-14T12:00:00",
    updateTime: "2025-12-14T12:00:00"
  }
]
```

**后端接口**: `GET /user-role/list/{userId}`  
**权限**: `admin:user-role:list`

---

### 2. 为用户分配角色（替换模式）

```javascript
import { assignRoles } from '@/api/user-role'

// 使用方式
await assignRoles({
  userId: 1,
  roleIds: [1, 2, 3]  // 会先删除用户原有的所有角色，再分配新角色
})

// 返回
{ 
  code: 200,
  message: "角色分配成功",
  data: "角色分配成功"
}
```

**后端接口**: `POST /user-role/assign`  
**权限**: `admin:user-role:assign`  
**说明**: 该接口会先删除用户原有的所有角色，再分配新角色（全量替换）

---

### 3. 为用户添加单个角色（追加模式）

```javascript
import { addUserRole } from '@/api/user-role'

// 使用方式
await addUserRole(userId, roleId)

// 返回
{ 
  code: 200,
  message: "角色添加成功",
  data: "角色添加成功"
}
```

**后端接口**: `POST /user-role/add/{userId}/{roleId}`  
**权限**: `admin:user-role:add`  
**说明**: 不删除原有角色，直接添加新角色

---

### 4. 删除用户的指定角色

```javascript
import { removeUserRole } from '@/api/user-role'

// 使用方式
await removeUserRole(userId, roleId)

// 返回
{ 
  code: 200,
  message: "角色删除成功",
  data: "角色删除成功"
}
```

**后端接口**: `DELETE /user-role/remove/{userId}/{roleId}`  
**权限**: `admin:user-role:remove`

---

### 5. 删除用户的所有角色

```javascript
import { removeAllUserRoles } from '@/api/user-role'

// 使用方式
await removeAllUserRoles(userId)

// 返回
{ 
  code: 200,
  message: "所有角色删除成功",
  data: "所有角色删除成功"
}
```

**后端接口**: `DELETE /user-role/remove-all/{userId}`  
**权限**: `admin:user-role:remove-all`

## 💡 核心特性

### 1. 双重加载机制

对话框打开时，并行加载：
- 用户当前角色列表
- 系统所有可用角色

```javascript
const [rolesRes, allRolesRes] = await Promise.all([
  getUserRoles(props.userData.id),
  getRoleList({ page: 1, pageSize: 1000 })
])
```

### 2. 智能角色过滤

已分配的角色不会出现在可选列表中：

```javascript
const availableRoles = computed(() => {
  return allRoles.value.filter(role => 
    !currentRoles.value.some(current => current.roleId === role.id)
  )
})
```

### 3. 实时预览

选择角色后立即显示角色详情表格：

```javascript
const selectedRoleDetails = computed(() => {
  return allRoles.value.filter(role => selectedRoleIds.value.includes(role.id))
})
```

### 4. 去重处理

提交时自动去重，避免重复分配：

```javascript
const uniqueRoleIds = [...new Set(allRoleIds)]
```

## 🐛 常见问题

### Q1: 为什么分配角色后列表没有更新？

**A**: 确保在 `handleRoleAssignSuccess` 回调中调用了 `loadUserList()` 来刷新列表。

### Q2: 如何处理角色加载失败？

**A**: 组件已内置错误处理，会显示 `ElMessage.error()` 提示。检查：
- 后端服务是否运行
- 网络连接是否正常
- 用户是否有相应权限

### Q3: 可以同时分配多个角色吗？

**A**: 可以。使用 `el-select` 的 `multiple` 属性支持多选，选择多个角色后点击"确定分配"即可。

### Q4: 如何只添加角色而不删除原有角色？

**A**: 使用 `addUserRole(userId, roleId)` API 而不是 `assignRoles()`。

## 📝 代码示例

### 完整的角色分配流程

```javascript
// 1. 导入 API
import { 
  getUserRoles, 
  assignRoles, 
  getRoleList 
} from '@/api/user-role'

// 2. 打开对话框
const handleAssignRole = (user) => {
  selectedUser.value = user
  roleDialogVisible.value = true
}

// 3. 在对话框中加载数据
const loadData = async () => {
  const [roles, allRoles] = await Promise.all([
    getUserRoles(userId),
    getRoleList({ page: 1, pageSize: 1000 })
  ])
  currentRoles.value = roles
  allRoles.value = allRoles.records
}

// 4. 分配角色
const handleSubmit = async () => {
  await assignRoles({
    userId: userData.id,
    roleIds: selectedRoleIds.value
  })
  ElMessage.success('分配成功')
  emit('success')
}

// 5. 刷新列表
const handleRoleAssignSuccess = () => {
  loadUserList()
}
```

## 🚀 性能优化

### 1. 并行加载

使用 `Promise.all()` 并行加载用户角色和所有角色：

```javascript
const [rolesRes, allRolesRes] = await Promise.all([
  getUserRoles(userId),
  getRoleList({ page: 1, pageSize: 1000 })
])
```

### 2. 计算属性缓存

使用 `computed` 自动缓存计算结果：

```javascript
const availableRoles = computed(() => {
  // 只在依赖变化时重新计算
  return allRoles.value.filter(...)
})
```

### 3. 去重处理

避免重复的角色分配：

```javascript
const uniqueRoleIds = [...new Set(allRoleIds)]
```
