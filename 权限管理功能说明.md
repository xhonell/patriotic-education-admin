# 权限管理功能说明

## 📋 功能概述

权限管理模块实现了系统菜单和权限的树形结构管理，支持懒加载和层级展示。

## 🎯 主要特性

### 1. 树形结构展示
- 使用 Element Plus 的树形表格组件
- 支持懒加载（lazy load）机制
- 父子级联关系清晰
- 可展开/收起子节点

### 2. 权限类型
- **菜单（type=1）**：系统菜单项，可以有子菜单，显示展开图标
- **按钮（type=2）**：操作权限（如增删改查按钮），不能有子级，不显示展开图标

### 3. 核心功能
- ✅ 权限列表查询（支持懒加载）
- ✅ 新增顶级权限
- ✅ 新增子权限
- ✅ 编辑权限信息
- ✅ 删除权限
- ✅ 状态开关（启用/禁用）
- ✅ 排序管理
- ✅ 搜索筛选（名称、类型、状态）

## 📊 数据结构

### 权限实体字段

```javascript
{
  id: Long,              // 主键
  name: String,          // 权限名称
  code: String,          // 权限标识（如 system:user:view）
  type: Byte,            // 权限类型（1 菜单 2 按钮/接口）
  path: String,          // 前端路由路径（菜单类型用）
  parentId: Long,        // 父级ID（0 为顶级）
  status: Boolean,       // 状态（true 启用，false 禁用）
  sort: Integer,         // 排序
  createTime: DateTime,  // 创建时间
  updateTime: DateTime   // 修改时间
}
```

## 🔌 API 接口

### 后端接口
```
基础路径: /admin/permission

GET  /list              - 获取权限列表（懒加载，无分页，返回 List）
GET  /select/{parentId} - 获取菜单下拉列表（懒加载，只返回启用的菜单）
POST /save              - 新增权限
POST /update            - 更新权限
POST /status/{id}       - 更新权限状态
DELETE /delete/{id}     - 删除权限
```

### 请求参数示例

#### 1. 获取权限列表（无分页）
```javascript
{
  parentId: 0,          // 父级ID（0 或不传为顶级，传具体ID为子级）
  name: "用户管理",      // 权限名称（可选）
  type: 1,              // 权限类型（可选）
  status: true          // 状态（可选）
}
```

#### 2. 获取菜单下拉树（完整树状结构）
```
GET /admin/permission/select

返回：List<PermissionTreeResponse>（树状结构，仅包含启用的菜单项）

树状结构说明：
- childrenPermission: 子权限列表
- 一次性返回完整树，无需多次请求
```

#### 3. 新增/更新权限
```javascript
{
  id: 1,                           // 更新时必填
  name: "用户管理",                 // 权限名称
  code: "system:user:view",        // 权限标识
  type: 1,                         // 权限类型
  path: "/system/user",            // 路由路径（菜单类型）
  parentId: 0,                     // 父级ID
  status: true,                    // 状态
  sort: 1                          // 排序
}
```

#### 4. 更新状态
```javascript
{
  status: true  // Boolean 类型
}
```

## 🎨 UI 特性

### 1. 表格展示
- 树形结构，清晰的层级关系
- 懒加载机制，按需加载子节点
- 菜单用蓝色标签，按钮用橙色标签
- 图标区分不同权限类型
- 状态开关，实时切换

### 2. 操作按钮
- **新增子项**：仅菜单类型显示，用于添加子权限（绿色圆形按钮）
- **编辑**：修改权限信息（橙色圆形按钮）
- **删除**：删除权限（红色圆形按钮）

### 3. 表单功能
- 权限名称（必填，2-50字符）
- 权限标识（必填，最多100字符）
- 权限类型（单选：菜单/按钮）
- 路由路径（仅菜单类型显示）
- 父级权限（**级联选择器，懒加载下拉**，不选则为顶级）
- 排序（数字，默认0）
- 状态（单选：启用/禁用）

### 4. 搜索筛选
- 按名称搜索
- 按类型筛选
- 按状态筛选
- 一键重置

## 💡 使用说明

### 1. 新增顶级菜单
1. 点击"新增权限"按钮
2. 填写权限名称，如"用户管理"
3. 填写权限标识，如"system:user"
4. 选择类型为"菜单"
5. 填写路由路径，如"/system/user"
6. **父级权限不选**（默认为顶级）
7. 设置排序和状态
8. 点击确定

### 2. 新增子权限（方式一：快捷按钮）
1. 在父级权限行，点击"新增子项"按钮（绿色加号）
2. 系统自动填充父级权限，且不可修改
3. 填写子权限信息
4. 选择类型（菜单或按钮）
5. 点击确定

### 3. 新增子权限（方式二：级联选择）
1. 点击"新增权限"按钮
2. 在父级权限字段，**点击级联选择器**
3. 懒加载展开并选择父级菜单
4. 填写其他信息
5. 点击确定

### 4. 编辑权限
1. 点击"编辑"按钮（橙色笔形图标）
2. 修改需要更改的信息
3. 点击确定保存

### 5. 删除权限
1. 点击"删除"按钮（红色删除图标）
2. 确认删除操作
3. 注意：删除父级会影响子级权限

### 6. 快速切换状态
- 直接点击状态开关，无需进入编辑页面
- 切换时显示loading状态
- 操作成功后显示提示消息

## 🔄 懒加载机制

### 表格树形懒加载
1. 初始加载时，只查询顶级权限（parentId=0）
2. 点击展开图标时，触发 `load` 方法
3. 自动传递当前行的 `id` 作为 `parentId` 查询子级
4. 子级数据动态加载并插入到表格中
5. 支持多级嵌套懒加载

### 级联选择器树状数据
1. 使用 `el-cascader` 组件，配置静态树状数据
2. 打开对话框时，调用 `/select` 接口一次性获取完整树
3. 仅显示启用状态的菜单项
4. 使用 `childrenPermission` 字段展示子级
5. 支持搜索和筛选功能
6. 减少请求次数，提升用户体验

### 优势
- 减少初始加载时间
- 降低服务器压力
- 按需加载数据（表格懒加载）
- 一次性获取完整树（级联选择器）
- 减少请求次数
- 级联选择更直观易用
- 自动过滤禁用项

## 🎯 权限标识规范

建议采用以下格式：
```
模块:功能:操作

示例：
- system:user:view      # 查看用户
- system:user:add       # 新增用户
- system:user:edit      # 编辑用户
- system:user:delete    # 删除用户
- content:article:list  # 文章列表
```

## 📝 注意事项

1. **无分页设计**：权限列表使用懒加载，无分页，直接展示树形结构
2. **父级选择**：使用级联选择器，一次性加载完整树状结构，更直观
3. **下拉接口**：`/select` 返回完整树状结构，仅包含启用的菜单项
4. **权限类型区分**：
   - 菜单类型（type=1）：可以有子级，显示展开图标
   - 按钮类型（type=2）：不能有子级，不显示展开图标
5. **路由路径** 仅菜单类型需要填写
6. **权限标识** 要保持唯一性
7. **删除操作** 需要确认是否有子权限
8. **排序值** 越小越靠前
9. **智能展开**：根据 type 字段自动判断是否显示展开图标
10. **级联选择** 支持搜索、筛选和可选任意层级

## 🔧 技术栈

- **前端框架**: Vue 3 + Composition API
- **UI 组件**: Element Plus
- **HTTP 请求**: Axios

- **表格组件**: el-table（树形+懒加载）
