# 用户角色分配 - 集成测试指南

## 🧪 测试环境准备

### 前置条件

- ✅ 后端服务运行在 `http://localhost:8801`
- ✅ 前端开发服务运行在 `http://localhost:3001`
- ✅ 已登录管理员账号
- ✅ 有 `admin:user-role:*` 权限

### 启动步骤

```bash
# 1. 启动后端服务
cd backend
mvn spring-boot:run

# 2. 启动前端开发服务
cd patriotic-education-admin
npm run dev

# 3. 打开浏览器
# http://localhost:3001
```

## 📋 测试用例

### 测试用例 1: 打开角色分配对话框

**目的**: 验证对话框能正确打开和加载数据

**步骤**:
1. 进入用户列表页面
2. 找到任意用户
3. 点击绿色的"分配角色"按钮

**预期结果**:
- ✅ 对话框正确打开
- ✅ 显示用户基本信息 (ID、用户名、邮箱)
- ✅ 加载用户当前角色列表
- ✅ 加载所有可用角色
- ✅ 没有加载错误提示

**验证方法**:
```javascript
// 在浏览器控制台查看
// 1. 打开 DevTools (F12)
// 2. 切换到 Network 标签
// 3. 查看两个 API 请求:
//    - GET /api/user-role/list/{userId}
//    - GET /api/admin/role/list
// 4. 都应该返回 200 状态码
```
