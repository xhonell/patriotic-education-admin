# 角色管理功能说明

## 📋 功能概述

角色管理是系统管理模块的核心功能之一，用于管理系统中的角色信息。角色是权限控制的重要组成部分，通过角色可以对用户进行权限分组管理。

## ✨ 功能特性

### 1. 角色列表
- 使用 Element Plus 的表格组件
- 支持分页展示
- 数据加载状态提示
- 响应式设计

### 2. 核心功能
- ✅ 角色列表查询（支持分页）
- ✅ 新增角色
- ✅ 编辑角色信息
- ✅ 删除角色
- ✅ 状态开关（启用/禁用）
- ✅ 搜索筛选（角色名称、状态）
- ✅ **权限分配**（树形选择器）

## 📊 数据结构

### 角色实体字段

```javascript
{
  id: Long,                    // 主键
  name: String,                // 角色名称
  description: String,         // 角色描述
  status: Boolean,             // 状态（true 启用，false 禁用）
  permissionIds: List<Long>,   // 权限ID列表
  createTime: LocalDateTime,   // 创建时间
  updateTime: LocalDateTime    // 更新时间
}
```

## 🔌 API 接口

### 1. 获取角色列表（分页）
```http
GET /admin/role/list
```

**请求参数**：
```javascript
{
  pageNum: Number,    // 页码
  pageSize: Number,   // 每页数量
  name: String,       // 角色名称（可选）
  status: Boolean     // 状态（可选）
}
```

**响应数据**：
```javascript
{
  code: 200,
  data: {
    total: Number,      // 总记录数
    list: [Role],       // 角色列表
    pageNum: Number,    // 当前页码
    pageSize: Number,   // 每页数量
    pages: Number       // 总页数
  }
}
```

### 2. 新增角色
```http
POST /admin/role/save
```

**请求体**：
```javascript
{
  name: String,            // 角色名称（必填）
  description: String,     // 角色描述（可选）
  status: Boolean,         // 状态（必填）
  permissionIds: [Long]    // 权限ID列表（可选）
}
```

### 3. 更新角色
```http
POST /admin/role/update
```

**请求体**：
```javascript
{
  id: Long,                // 角色ID（必填）
  name: String,            // 角色名称（必填）
  description: String,     // 角色描述（可选）
  status: Boolean,         // 状态（必填）
  permissionIds: [Long]    // 权限ID列表（可选）
}
```

### 4. 修改角色状态
```http
POST /admin/role/status/{id}
```

**路径参数**：
- `id`: 角色ID

**请求体**：
```javascript
{
  status: Boolean    // 新状态
}
```

### 5. 删除角色
```http
DELETE /admin/role/delete/{id}
```

**路径参数**：
- `id`: 角色ID

## 🎨 页面功能

### 搜索区域
1. **角色名称**：支持模糊搜索
2. **状态筛选**：可选择启用/禁用
3. **搜索按钮**：执行搜索操作
4. **重置按钮**：清空搜索条件

### 数据表格
| 列名 | 说明 | 宽度 | 备注 |
|-----|------|------|------|
| 序号 | 自动编号 | 60px | - |
| 角色名称 | 显示角色名称 | 自适应 | 支持溢出省略 |
| 角色描述 | 显示角色描述 | 自适应 | 支持溢出省略，空值显示"-" |
| 状态 | 开关按钮 | 90px | 支持快速切换 |
| 创建时间 | 格式化显示 | 160px | yyyy-MM-dd HH:mm:ss |
| 更新时间 | 格式化显示 | 160px | yyyy-MM-dd HH:mm:ss |
| 操作 | 编辑、删除 | 180px | 固定在右侧 |

### 状态开关
- **启用状态**：蓝色开关
- **禁用状态**：灰色开关
- **加载状态**：显示 loading 图标
- **即时生效**：点击后立即更新状态
- **错误恢复**：失败时自动刷新列表恢复原状态

### 新增/编辑对话框

**表单字段**：

1. **角色名称**
   - 必填项
   - 长度限制：2-50 个字符
   - 带字数统计
   - 图标提示

2. **角色描述**
   - 可选项
   - 多行文本输入
   - 长度限制：最多 200 个字符
   - 带字数统计

3. **状态**
   - 必填项
   - 单选按钮组
   - 默认：启用
   - 选项：启用/禁用

### 删除操作
- 二次确认弹窗
- 显示被删除的角色名称
- 确认后执行删除
- 支持取消操作
- 智能分页：最后一页最后一条删除后自动返回上一页

## 📱 交互体验

### 1. 加载状态
- 表格加载：显示骨架屏
- 状态切换：按钮 loading 效果
- 表单提交：按钮 loading 效果

### 2. 消息提示
- ✅ 操作成功：绿色成功提示
- ❌ 操作失败：红色错误提示
- ⚠️ 删除确认：警告确认框
- 📝 表单验证：实时错误提示

### 3. 表单验证
```javascript
{
  name: [
    { required: true, message: '请输入角色名称', trigger: 'blur' },
    { min: 2, max: 50, message: '角色名称长度在 2 到 50 个字符', trigger: 'blur' }
  ],
  description: [
    { max: 200, message: '角色描述不能超过 200 个字符', trigger: 'blur' }
  ],
  status: [
    { required: true, message: '请选择状态', trigger: 'change' }
  ]
}
```

## 📝 注意事项

1. **角色名称唯一性**：建议后端进行唯一性校验
2. **角色描述可选**：可以为空
3. **状态默认值**：新增时默认为启用状态
4. **删除限制**：建议检查是否有用户关联该角色
5. **权限关联**：删除角色时需要考虑权限关联关系
6. **分页设置**：默认每页 10 条，可选 10/20/50/100
7. **搜索重置**：重置后返回第一页
8. **时间格式**：使用 LocalDateTime，前端格式化显示

## 🎯 典型场景

### 场景 1：创建管理员角色
```
1. 点击"新增角色"按钮
2. 填写：
   - 角色名称：系统管理员
   - 角色描述：拥有系统所有权限的管理员角色
   - 状态：启用
3. 点击"确定"保存
4. 刷新列表，新角色出现在列表中
```

### 场景 2：禁用角色
```
1. 在列表中找到目标角色
2. 点击状态开关
3. 系统自动将状态改为禁用
4. 开关变为灰色
5. 提示"已禁用"
```

### 场景 3：编辑角色信息
```
1. 点击角色行的"编辑"按钮
2. 弹出编辑对话框，自动填充当前数据
3. 修改角色描述
4. 点击"确定"保存
5. 列表自动刷新显示新数据
```

### 场景 4：删除角色
```
1. 点击角色行的"删除"按钮
2. 弹出确认框："确定要删除角色"XXX"吗？"
3. 点击"确定"
4. 系统删除角色
5. 提示"删除成功"
6. 列表自动刷新
```

### 场景 5：搜索角色
```
1. 在搜索框输入角色名称关键字
2. 选择状态：启用/禁用/全部
3. 点击"搜索"按钮
4. 表格显示符合条件的结果
5. 分页器更新总数
```

## 🔧 技术栈

- **前端框架**: Vue 3 + Composition API
- **UI 组件**: Element Plus
- **图标**: Element Plus Icons
- **HTTP 请求**: Axios（封装在 request.js）
- **路由**: Vue Router

## 🚀 路由配置

```javascript
{
  path: '/system',
  component: Layout,
  redirect: '/system/settings',
  meta: { title: '系统管理', icon: 'Tools' },
  children: [
    {
      path: 'role',
      name: 'RoleManagement',
      component: () => import('@/views/content/role.vue'),
      meta: { title: '角色管理', icon: 'UserFilled' }
    }
  ]
}
```

访问路径：`/system/role`
